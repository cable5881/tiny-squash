# TinySquash 更新日志

记录每次版本更新的功能变更。在每次合入 `main` 分支前更新此文件。

---

## 2026-04-04 (v1.3)

### 修复
- **管理员后台无权访问**：旧版 `users` 表缺少 `plan`/`plan_expires_at`/`preferences` 列导致 `/api/auth/me` 查询失败。`initTables()` 现在会执行 `ALTER TABLE ... ADD COLUMN` 迁移，对已存在列自动忽略。

### 新增
- **套餐配置管理**（管理后台）：
  - 新增 `plan_configs` 数据表，存储套餐权益配置（价格、每日限额、文件数、文件大小、格式、ZIP 下载等）
  - 管理后台新增"套餐权益配置"面板，可直接编辑各套餐参数并即时生效
  - 管理 API `GET/PUT /api/admin/plans` — 套餐配置 CRUD
  - 公开 API `GET /api/plans` — 前端获取当前生效套餐配置
  - 定价页和配额系统从 DB 动态读取配置，支持硬编码兜底
  - 用户列表新增"套餐"列显示
- **定价页全新设计**：
  - 居中排版，渐变 Hero 区域，简洁视觉层次
  - 功能列表使用圆形图标和删除线区分可用/不可用
  - 新增三套餐横向对比表格
  - 从 `/api/plans` 动态加载配置，管理后台修改即时生效
  - 响应式布局优化（移动端单列）

### 改进
- 管理后台 UI 重构，新增套餐配置区域置顶
- CSS 新增管理配置面板、定价卡片、对比表格等组件样式

## 2026-04-04 (v1.2)

### 新增
- **用户体系与配额系统**：
  - 游客每天 3 次免费压缩，Free 用户 20 次/天，Pro 用户无限
  - 角色权限矩阵：游客/Free/Pro/Admin，功能逐级解锁
  - 配额检查 API (`/api/usage/check`) 和用量上报 API (`/api/usage/record`)
  - `usage_daily` 表追踪每日用量，`compress_logs` 表记录压缩历史
- **定价页面** (`/pricing.html`)：
  - 游客 / Free / Pro 三列定价卡片，支持月付/年付切换
  - 8 条 FAQ 常见问题（隐私、退款、格式、配额等）
- **个人中心** (`/profile.html`)：
  - 用户信息卡片（头像/昵称/邮箱/角色/Plan/注册时间）
  - 使用统计面板（总压缩次数/节省空间/访问次数/今日剩余额度）
  - 压缩历史记录（分页表格，支持清除）
  - 账户操作（编辑昵称/退出登录/删除账户）
- **升级转化弹窗**：配额用尽/文件数超限/文件过大/使用 Pro 功能时自动弹出
- **首页升级横幅**：非 Pro 用户显示升级引导条
- **首页配额提示条**：实时显示今日剩余压缩次数
- **AVIF 格式支持**（Pro 专属）
- **导航栏定价入口**：首页 header 新增"定价"链接
- **头像/昵称可点击进入个人中心**

### API 新增
- `POST /api/usage/check` — 检查今日剩余配额
- `POST /api/usage/record` — 上报压缩用量 + 写入日志
- `GET /api/user/profile` — 获取个人信息 + 统计
- `PUT /api/user/profile` — 更新昵称
- `GET/PUT /api/user/preferences` — 偏好设置（Pro 专属）
- `GET /api/user/compress-logs` — 分页查询压缩历史
- `DELETE /api/user/compress-logs` — 清除压缩历史
- `DELETE /api/user/account` — 删除账户及关联数据
- `GET /api/auth/me` 扩展返回 `plan` / `limits` 字段，游客也返回 plan 信息

### 改进
- `app.js` 全面重构：上传/压缩/下载流程集成配额检查和用量上报
- `store.js` 状态扩展 plan/limits 字段
- `style.css` 大幅扩展：新增 20+ 组件样式
